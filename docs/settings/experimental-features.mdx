# Turning On Experimental Features

**Experimental** (in Workspace Settings) is where an admin can switch on **beta features** for the workspace before they're generally available. Turning one on enables it for **every member** of the current workspace.

It's the "try new things early" switchboard — with the understanding that beta features are still in progress.

## Before You Start

1. ✅ **You're an admin** — this page needs the **workspace settings** permission (`workspace.settings.write`). Members without it don't see it, and the server enforces the same rule.
2. ✅ **You're in a team workspace** — the page isn't available in a Personal workspace.

## Step 1 — Open Experimental

Go to **Workspace Settings** and open **Experimental** (under **Labs** in the settings sidebar).

You'll see the heading _"Experimental features"_ and the note: _"Try features still in beta. Turning one on enables it for your whole workspace — they may change, break, or disappear."_

## Step 2 — Turn a Feature On or Off

Each available beta feature is listed as its own row with:

- Its **name** and an **Experimental** badge
- A short **description** (when provided)
- A **toggle** to enable or disable it

Flip a toggle to opt the workspace **in** or **out** of that feature. The change applies **immediately** to every member of the workspace.

> **Heads-up:** Experimental features are still in beta — they **may change, break, or disappear** without notice. Turn them on knowing they're not final.

## When the List Is Empty

If there are no beta features being offered right now, the page shows:

> _"No experimental features available right now."_

This is normal — the list is driven by whatever features frndOS is currently trialing. New ones appear here as they open up.

## How This Affects Members

- Enabling a feature here turns it on for the **whole workspace** — every member gets it, not just you.
- There's no per-member opt-in on this page; it's a workspace-wide switch.

## FAQ

**Q: I don't see the Experimental page.**
It's admin-only (needs workspace settings permission) and team-only. If you're not an admin, or you're in a Personal workspace, it won't appear. Ask a workspace admin.

**Q: The list is empty.**
That just means no beta features are being offered right now. The list changes as frndOS trials new features — check back later.

**Q: If I turn a feature on, who gets it?**
Everyone in the workspace. It's a workspace-wide toggle, not a personal one.

**Q: Are experimental features safe to rely on?**
Treat them as beta. They can change, break, or be removed at any time — don't build critical work around a feature that's still experimental.

**Q: Do my changes need saving?**
No — each toggle applies immediately.
