# CLAUDE.md

This file provides guidance to Claude Code (claude.ai/code) when working with code in this repository.

## Project Overview

**frndos-docs** (`frnd-docs`) is the **frndOS Help Center** — self-service product guidance for every frndOS end user. It is a standalone project inside the `frnd-github` monorepo with its own `.git`, dependency management, and Vercel deployment.

Built with **Docusaurus 3** + **TinaCMS** so non-technical content authors can edit guides and changelog entries through a browser-based visual editor.

- **Guides** — help articles organized by frndOS module (`docs/`)
- **What's New** — changelog blog for product updates (`blog/`)
- **Search** — local full-text search across guides and changelog
- **TinaCMS** — visual editor at `/admin` for content authors

## Development Commands

```bash
# Install (exact, from lockfile)
npm ci

# Dev server (Docusaurus + TinaCMS, local content mode)
npm run dev:local          # → http://localhost:3333  (editor: /admin)

# One-shot production build with TinaCMS local mode
npm run build-local

# Smart build wrapper (what Vercel runs via `npm run build`)
npm run build              # build:cloud if Tina creds present, else build:local

# Content author workflow: stage + commit + push → Vercel redeploy
npm run publish

# Generate translation skeleton (see Bilingual note below)
npm run write-translations -- --locale en
```

> TinaCMS build is memory-hungry. The npm scripts already set `NODE_OPTIONS=--max-old-space-size=8192`; bump higher if you hit a Node OOM.

## Key Conventions

### Sidebar is auto-generated
`sidebars.js` uses `{ type: "autogenerated", dirName: "." }`. The sidebar is built entirely from the `docs/` folder structure:
- **Category** label / emoji / position → `_category_.json` inside the folder.
- **Doc** position within a category → `sidebar_position` in the doc's frontmatter.

`config/sidebar/index.json` is **legacy** — kept on disk as historical reference only, not wired in. Its paths are stale (`brand-settings/`, `creative-brief/`, `account-settings/` no longer exist). Do not edit it expecting effect; edit folder structure + `_category_.json` instead.

### Site config flows through JSON, not the .js file
`docusaurus.config.js` reads `config/docusaurus/index.json` for title, navbar, footer, and logo. To change site chrome, edit the **JSON** (which TinaCMS also manages), not hardcoded values in the `.js`.

### TinaCMS collections (`tina/config.jsx`)
| Collection | `path` | Purpose |
|------------|--------|---------|
| Post — "What's New (Changelog)" | `blog` | Changelog entries |
| Doc — "Help Articles" | `docs` | Guide articles; has a `frndOS Module` select matching the 11 category folders |
| Settings | `config/docusaurus` | Site title/navbar/footer/logo |
| Homepage | `config/homepage` | Landing blocks |
| Pages | `src/pages` | MDX pages |
| Sidebar | `config/sidebar` | Legacy, unused |

When adding a new module category, keep the folder name, `_category_.json` label, and the `frndOS Module` select option in `tina/config.jsx` in sync.

### Adding content
- **New guide:** create `docs/<module>/<slug>.mdx` with frontmatter (`title`, `description`, `sidebar_position`). Use a relative link like `./other-doc` for cross-references.
- **New category:** create `docs/<module>/` + `_category_.json` (`label`, `position`, `link.type: "generated-index"`), and add the matching option to the `frndOS Module` select in `tina/config.jsx`.
- **New changelog entry:** create `blog/YYYY-MM-DD-<slug>.mdx`, reference an author key from `blog/authors.yml`, and put a `<!-- truncate -->` after the summary line.

### Image assets — no spaces / unsafe chars in names
Docusaurus' mdx-loader resolves **absolute** image paths (`/img/...`) without `decodeURIComponent`, so a space in a folder or file name becomes `%20` and the loader fails with `Image ... not found` — breaking the build. TinaCMS lets authors create spaced folders / upload spaced filenames (e.g. `DEV local/alva logo.jpeg`), and dragged macOS screenshots (`Screenshot at ....png`) hit the same trap.

Guard: `scripts/normalize-assets.js` walks `static/img/`, renames any unsafe segment to a build-safe slug (spaces → `-`, case & camelCase preserved), and rewrites the matching refs in `docs/` + `blog/`. It runs automatically via `predev` / `predev:local` / `prebuild` (npm pre-hooks) and is idempotent. To run manually: `npm run normalize-assets`. Keep asset names URL-safe to begin with and this is a no-op.

### Bilingual status (IMPORTANT — do not assume i18n works)
The README describes a bilingual ID/EN setup with default locale `id`, but the **actual** `docusaurus.config.js` ships `locales: ["en"]`, `defaultLocale: "en"`, and there is **no `i18n/` folder**. The site is currently **English-only**. If asked to add Indonesian, the config and an `i18n/` tree must be created from scratch — don't presume existing scaffolding.

## Environment

`.env` holds TinaCloud credentials (gitignored):
```
TINA_TOKEN=
NEXT_PUBLIC_TINA_CLIENT_ID=
```
Without both, builds fall back to `build:local` (admin is built but read-only in production — local-only editing pattern). `tina/__generated__` and `static/admin` are generated and gitignored.

## Deployment

- Hosted on **Vercel**, wired to the GitHub repo. A push to the tracked branch triggers a redeploy.
- `npm run publish` is the content-author path: it shows the diff, prompts for a commit message (defaults to `Content update <date>`), stages everything, commits, and pushes. It refuses on a clean tree or detached HEAD and never force-pushes.
- Vercel runs `npm run build` → `scripts/build.js` picks `build:cloud` (Tina creds present) or `build:local`.

It reads real feature behavior from the `frnd-web` codebase, marks anything unreadable (third-party iframes, retired routes) as `TODO` instead of guessing, and validates with a build. Invoke it before writing docs from a keyword.

## Rules

- No AI attribution in commit messages (no co-author, no "Generated with Claude Code").
- Communicate in Bahasa Indonesia (keep proper nouns / technical terms untranslated) — per monorepo `.claude/rules/language.md`.
- `onBrokenLinks: "throw"` — broken internal links fail the build. Verify cross-references before considering a content change done.
- Content prose is end-user facing; match the existing concise, task-oriented voice of neighboring articles.
