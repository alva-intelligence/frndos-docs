# AGENTS.md — Developer Guide for frndos-docs

## Overview

**frndos-docs** is the **frndOS Help Center** — a Docusaurus 3 + TinaCMS site that serves self-service guides and a "What's New" changelog to frndOS end users. Standalone project in the `frnd-github` monorepo (own `.git`, own deploy on Vercel). Content authors edit through the TinaCMS visual editor at `/admin`; engineers edit MDX/config directly.

## Commands

### Development

```bash
npm ci                  # install from lockfile (Node 22+)
npm run dev:local       # dev server → http://localhost:3333 (editor at /admin)
npm run clear           # clear Docusaurus cache if dev gets stuck
```

### Build

```bash
npm run build-local     # one-shot local build (Tina local mode)
npm run build           # smart wrapper Vercel uses (cloud if creds, else local)
npm run serve           # serve a production build on :3333
```

Builds set `NODE_OPTIONS=--max-old-space-size=8192` (Tina is memory-heavy). Increase on OOM.

### Publish (content authors)

```bash
npm run publish         # diff → prompt commit msg → add -A → commit → push → Vercel redeploy
```

### Testing

No test framework. Validation is the build itself: `onBrokenLinks: "throw"` means broken internal links fail `npm run build`. Run a build before declaring a content/structure change done.

## Architecture

- **Docusaurus 3.3.2** classic preset, **React 18**, MDX content.
- **TinaCMS 3.8** overlays a Git-backed CMS. Schema lives in `tina/config.jsx`; generated artifacts in `tina/__generated__` (gitignored).
- **Search** via `@easyops-cn/docusaurus-search-local` (indexes docs + blog).
- `docusaurus.config.js` is data-driven: it reads `config/docusaurus/index.json` for title/navbar/footer/logo. Edit the JSON, not the JS.
- `/` redirects to `/docs/intro` (`src/pages/index.js`).

### Directory map

```
docs/<module>/*.mdx        # guides; folder = module, _category_.json = sidebar category
docs/<module>/_category_.json
blog/YYYY-MM-DD-<slug>.mdx  # changelog; authors.yml defines author keys
config/{docusaurus,homepage,sidebar}/index.json   # Tina-managed JSON (sidebar = legacy/unused)
src/components/{Hero,Features,YouTubeEmbed}/       # custom Tina blocks (+ template.* schema)
src/theme/                 # MDXComponents + Tina template registry
tina/config.jsx            # TinaCMS collections + fields
sidebars.js                # autogenerated from docs/ folder tree
scripts/{build,publish}.js
```

Module folders: `getting-started`, `brand-setup`, `studio`, `insights`, `research`, `askfrnd`, `growth`, `collaboration`, `projects`, `workspace`, `settings`.

## Conventions

- **Sidebar is autogenerated** from `docs/` structure. Category metadata → `_category_.json`; doc order → `sidebar_position` frontmatter. `config/sidebar/index.json` is legacy and unused (stale paths) — do not edit it.
- **Adding a module category:** new `docs/<module>/` folder + `_category_.json`, and add the matching option to the `frndOS Module` select in `tina/config.jsx`. Keep folder name / category label / select option in sync.
- **New guide:** `docs/<module>/<slug>.mdx` with `title`, `description`, `sidebar_position`; relative links (`./sibling`) for cross-refs.
- **New changelog:** `blog/YYYY-MM-DD-<slug>.mdx`, `authors:` key from `authors.yml`, `<!-- truncate -->` after the summary.
- **Site chrome** (title/nav/footer/logo) lives in `config/docusaurus/index.json`.
- Match the existing concise, task-oriented, end-user voice in neighboring articles.

## Gotchas

- **Not bilingual yet.** README claims ID/EN with default `id`, but config is `locales: ["en"]`, default `en`, and there is **no `i18n/` folder**. Site is English-only today. Adding ID means building the i18n config + tree from scratch.
- **Legacy `config/sidebar/index.json`** references folders that no longer exist (`brand-settings/`, `creative-brief/`, `account-settings/`). Ignore it.
- **Package manager is npm**, even though `.yarnrc.yml` and `.npmrc` (`engine-strict=true`) are present. Use `npm ci` / `npm run …`.
- `tina/__generated__` and `static/admin` are generated — never hand-edit, never commit (gitignored).
- Without TinaCloud creds in `.env`, the `/admin` editor is read-only in production (local-editing pattern).

## Skills

- **`writing-help-docs-from-code`** (`.claude/skills/`) — given a frndOS feature keyword, generates/updates Help Center guides grounded in the `frnd-web` codebase. Reads real UI/behavior from code, marks unreadable parts (vendor iframes, retired routes) as `TODO` rather than guessing, and validates with a build. Use it whenever docs are requested from just a keyword.

## Rules

- No AI attribution in commit messages.
- Communicate in Bahasa Indonesia (proper nouns / technical terms stay original) — monorepo `.claude/rules/language.md`.
- Don't add/remove dependencies without user approval.
- Broken internal links break the build — verify cross-references.
